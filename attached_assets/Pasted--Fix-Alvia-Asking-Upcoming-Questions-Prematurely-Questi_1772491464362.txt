 Fix: Alvia Asking Upcoming Questions Prematurely (Question Loop Bug)

 Context

 A tester (Phil) reported questions got stuck in a repetitive loop. Alvia asked all 5 template questions while the system was still on questionIndex: 0 — because Phil verbally said "next question" and Alvia could see the full question list in her prompt. The system's currentQuestionIndex only advances when the respondent clicks the Next Question button, so when the system later advanced through indices 1-5, every question was re-asked. Phil said "You already asked me that" three times.

 Root Cause

 1. The suggest_next_question runtime message (voice-interview.ts:2363) literally scripts Alvia to say "shall we move to the next question?" — this invites the respondent to verbally agree, then Alvia reads the next question from her prompt
 2. Instruction #11 says "respondent will click the button" but doesn't tell Alvia not to read ahead
 3. No repeat-awareness instruction — when Alvia does end up on a question she's already covered, there's no  instruction telling her to acknowledge that

 Approach: Moderated Nudge

 Minimal, targeted changes — not a prompt restructure. Fix the one line that actively causes the problem, add one sentence to an existing instruction, and add one new instruction as a safety net.

 Changes

 A. server/voice-interview.ts — fix the trigger (line 2362-2363)

 This is the single highest-impact change. It's the exact line that scripts Alvia to invite a verbal "next question" response.

 Before:
 "Based on the conversation so far, the respondent has provided a comprehensive answer to this question. When there's a natural pause or you finish responding to their latest point, warmly offer to move on. You might say something like: 'Thank you for sharing that. Is there anything else you'd like to add, or shall we move to the next question?' Wait for their response - they will click the Next Question button when ready."

 After:
 "Based on the conversation so far, the respondent has provided a comprehensive answer to this question. When there's a natural pause, thank them for their answer and let them know they can click the Next Question button below whenever they're ready to move on."

 B. server/voice-interview/instructions.ts — extend instruction #11, add #13

 Applied consistently across all three builders: buildInterviewInstructions (line 88),
 buildResumeInstructions (line 331), buildRefreshInstructions (line 364).

 Extend instruction #11

 Before:
 11. The respondent will click the Next Question button when ready to move on. You can refer to this button as "the Next Question button below" if appropriate.

 After:
 11. The respondent will click the Next Question button when ready to move on. You can refer to this button as "the Next Question button below" if appropriate. Do not ask upcoming questions yourself as the system needs to advance them.

 Add instruction #13 (repeat awareness)

 Add after the existing instruction #12 in all three builders:

 13. If you think a topic has already been discussed earlier in the conversation, briefly acknowledge that before exploring it (e.g. "I think you touched on this earlier — let me build on that") rather than asking from scratch.

 Fix resume instruction #2 (line 318-319)

 Before: Ask if they'd like to add anything or move to the next question.
 After: Ask if they'd like to add anything, or let them know they can click the Next Question button below to move on.

 C. server/barbara-orchestrator.ts — advisory guardrail (after line 381)

 Add one new responsibility rule after the existing QUESTION DEDUPLICATION rule:

 5. QUESTION TEXT CONFIDENTIALITY: Do not quote or preview the text of any UPCOMING QUESTION in your guidance message. Your guidance goes directly into Alvia's prompt. Reference upcoming questions by number (e.g., "Q3 will cover this") if needed, not by content.

 Renumber existing rules 5-6 to 6-7.
