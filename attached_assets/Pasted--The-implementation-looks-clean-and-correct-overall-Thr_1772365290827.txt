Bug: Circular import in llm-config.ts (real issue)                                                                                                                     
                                                                                                                                                                            
  llm-config.ts:1 imports log from ./index. Meanwhile index.ts:7 imports from ./llm-config. This is a circular dependency. It works here only because by the time

  The string "https://api.openai.com/v1 (default)" looks like a URL with a suffix baked in. If someone copies it from logs, they'd get a bad URL. Better to separate the

  const openaiBaseUrl = process.env.OPENAI_BASE_URL;
  log(`[OpenAI] Base URL: ${openaiBaseUrl || "https://api.openai.com/v1"} ${openaiBaseUrl ? "" : "(default)"}`, ...);
